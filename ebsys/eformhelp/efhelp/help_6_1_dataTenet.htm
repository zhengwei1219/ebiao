1. 为什么要用数据集
    一般来说，数据都是保存在后台数据库中，这是数据库这一层，另外表单界面主要由一些控件组成，在这些控件上输入数据到数据库，或是从数据库中查出数据到这些控件上来显示，表单上的这些控件常称之为界面层。表单主要是处理这两层之间的数据交换，如只有这两层，那么在处理一些复杂的数据转换存储等功能时就被迫需要定制特别的代码来实现，程序的灵活性，易用性和可扩展性都会较差。所以需要在这两层中间加上一层：数据集层，数据集层的表示形式就是一个数据集控件，它用于在数据库层和界面层之间中转数据，作用非常大，它设计的好坏直接关系到表单工具的价值。使用数据集可以使数据与样式分离，数据保存在数据集中，样式是由界面层的控件组成。一般来说，一个表单界面上都会要用到一个或多个数据集控件。界面上的数据都是通过数据集与后台进行交互,界面上的值先存到数据集上,数据集提交到数据库,反之从数据库中执行的一些sql语句查询到的数据生成xml串,传到前台的数据集上,然后数据集把数据分发到各个控件上.

    数据集本身的数据是一个XML格式的数据，其结构见上一节：数据集的XML数据结构，可以由dataset1.xml 命令来得到，它保存了一个数据集的完整数据，是一个二维的表格状数据。 dataset是表单开发的数据层,后台数据库中的数据和前台界面上的控件之间通过dataset来进行数据交互.读数据时,执行数据集属性窗口中的SQL语句,其运行结果以XML串方式来初始化数据集,由数据集将数据分发到它所绑定的控件上.这样便实现了将数据库中的数据读出到界面上的过程了.写数据时,先将数据集所绑定的控件上的数据保存到数据集中,即用数据集来收集表单上所有要保存的数据,然后将一个表单的所有数据集的数据提交到后台,在后台将数据转换成SQL命令,然后再执行这些SQL命令保存数据到数据库中.这样便实现 了将界面上的输入数据保存到数据库中了.
3. dataset的数据同名字段传递规则
    当数据集打开时（比如用ds.Open()命令），一般是运行数据集的SQL语句，将SQL语句的运行结果当作本数据集的数据（是数据集的XML数据，可以通过ds.xml来取到）。数据集的SQL语句的SELECT节会有一些字段名，这些字段名不需要和数据集的字段名（在数据集属性窗口的数据项中定义）一一对应。即数据集的SQL语句输出的字段名有可能在数据集的字段名中存在，也可能不存在。比如下图中数据集的SQL语句为：select F1,F3,F5,F7 from t1 ，即输出的字段名为：F1,F3,F5,F7这四个字段。而数据集的字段名为：F1,F2,F3,F4这四个字段。其中只有F1,F3这两个字段是两边都有的。根据字段同名来传递数据的规则，在数据集打开时只会将 select F1,F3,F5,F7 from t1 运行结果中的 F1,F3 这两个字段的数据传递到数据集中。数据集中的F2,F4字段的值就是空着的（或为默认值）。

    当保存时，一般需要将数据集中的数据（是数据集的XML数据，可以通过ds.xml来取到）保存到数据集的保存表名（可以通过ds.savetable来存取）处指定的物理表中。数据集的字段名和要保存的物理表的字段名也同样不需要一一对应。 保存时只会将数据集和物理表中同时存在的字段保存。如图所示数据集中的字段为F1,F2,F3,F4这四个字段，而要保存的物理表的字段名为：F1,F2,F5,F6这四个字段。此时只会将数据集中的F1,F2这两个字段的数据保存。 因为使用了同名字段传递数据的规则，使得在设置数据集的SQL语句时常常可以直接写上: select * from table1 之类的SQL语句。因为SQL语句可以通过字段别名的方式（如： select f1 as othername …）来改变字段名，而数据集中的字段名也是可以随意修改的，只有数据集的保存表名属性中的物理表的字段名是无法修改的。这样就可以很方便地通过改变字段名的方式，控制读出哪些字段的数据，以及保存哪些字段的数据。
4. dataset的数据控制结构

    数据集中的数据用xml属性来取得,即ds.xml便可取得整个数据集的数据,它是XML格式的,包含字段信息.也可以用ds.oDom来取得数据集中数据的dom对象. 数据集绑定到界面上的控件分两类,一个是grid控件,另一个是其它零散控件,如text,radio,dropdownlist等.数据集中还含有一个当前行记录的数组，表示数据集的当前行数据，固定最多只有一条记录，刚打开数据集时是指第一条记录。它是由一个javascript的数组来保存的。可用 dataset1.Field(‘字段名’).Value 命令来存取它的值，数据集中的数据(指 ds.xml)先取出当前行记录到这个记录当前行的数组中,可以用ds.MoveFirst() ds.MovePrev() ds.MoveNext() ds.MoveLast() ds.SetPos(pos) 等方法来移动当前行记录, 每次记录移动时都会改变它的值,把当前行记录回写到数据集中时用 dataset1.Update() 命令。表格(grid)控件，用数据集的 dataset1.dset_cont() 命令直接将整个数据集的数据都放入到表格控件中。用 dataset1.fset_cont() 命令将最新的当前行记录刷新表格控件的当前行记录，用 dataset1.cont_onDataChange() 命令将表格控件上改动在当前行记录回写到数据集的当前行记录中。零散控件，如编辑框，combobox等，它是相对表格控件而言的，指非表格控件。因为它们的机制差不多，所以就在一起说明。用 dataset1.fset_cont1() 命令用数据集的最新的当前行记录刷新此数据集所绑定的所有零散控件，用 dataset1.cont1_fset() 命令则是将绑定的零散控件上的变动后的值回写到数据集的当前行记录中。
5. dataset的数据流向
     数据集的XML结构的数据 --> 数据集的当前行记录 --> 数据集所绑定的零散控件上，反向也是如此。
     数据集的XML结构的数据 --> 数据集的当前行记录 --> 数据集所绑定的表格控件上，反向也是如此。
     数据集的XML结构的数据 --> 数据集所绑定的表格控件上。
6. 数据集控件与其它控件交互
     在下面的例子中通过dataset1.OpenXmlData(xml数据)和dataset2.Open(sql语句)给数据集装入数据.
6.1.1 数据集的OpenXmlData方法给数据集装入数据
    
    我们把画红圈的text控件与数据集绑定，在自定义函数中通过dataset1.OpenXmlData("xml串")给数据集控件的xml数据区装入数据.dataset1.dset_fset()将dataset的当前行记录传给当前行数组中，通过dataset1.fset_cont1()将field字段集传递给text控件.你修改text控件的之后触发了onchange事件,将零散控件上的当前行的值保存到数据集的当前行数组中.它是通过执行 ds.cont1_fset()方法,将零散控件上的当前行的值保存到数据集的当前行数组中,并执行 ds.cont_onDataChange()方法.

6.2 其它控件绑定数据集什么事件传值
    textarea控件:当用户在textarea控件中修改了数据，就触发了onchange事件把值传到数据集上.
    combobox控件:当用户在combobox控件中修改了数据，就触发了onchange事件把值传到数据集上.
    listbox控件:当用户在listbox控件中修改了数据，就触发了onchange事件把值传到数据集上.
    dropdownlist:当用户在控件在dropdownlist控件中修改了数据，就触发了onchange事件把值传到数据集上.
    checkbox控件:当用户在checkbox控件上单击时触发了onclick事件把值传到数据集上.
    checkboxlist控件:当用户在checkboxlist控件中单击时触发onclick事件中传值.
    radiolist控件:当用户在radiolist控件上单击的时候触发onclick事件中传值.
    grid控件:用在grid控件上修改数据时,触发在onDataChange事件传值到数据集上.
